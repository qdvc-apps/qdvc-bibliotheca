- `config.py` — validated `ui_backend` key. The selector appears in **both** UIs' Preferences (so a GTK3 user can switch forward and vice-versa); it takes effect on restart.
- `ui_prefs.py` (pure) — extracted the toolkit-independent logic both windows now share: J-Flag presets, sort-spec load/save/describe, citation-style get/store, validation-report formatting, the shortcuts table.

**The GTK4 UI (HIG-driven)**
- `Adw.Application` + accelerator registration; `Adw.ApplicationWindow` with an `Adw.ViewStack`/`ViewSwitcher` for the four views, a single header bar + primary hamburger menu (Preferences/Shortcuts/About last), and a status line.
- A `Gio.Action` command layer (`win.*`) replacing per-widget handlers.
- Catalogue as `Adw.OverlaySplitView` + `Gtk.ColumnView` (8 columns, markup Outlet) + `Gtk.TreeListModel` sidebar with counts and work-management, Pane-3 detail with a CSL `Gtk.DropDown` and highlighted notes, and `Gtk.PopoverMenu` context menus.
- Authors/Outlets/DOI views, and every dialog rewritten async (`Adw.MessageDialog`/`Gtk.FileDialog`/callback-bearing `Adw.Window`s) — import, sort, allocate, my-works editor, preferences (`Adw.PreferencesWindow`, live-apply), and shortcuts.

**Docs** — a new `MAINTENANCE_GTK3_GTK4.md` element-by-element comparison, plus updated `MAINTENANCE.md` (tree, requirements, dispatcher §3.3, config keys, maintenance tasks) and `README.md`.

All functionality is retained; only access paths changed where the HIG differs (menubar/toolbar → header bar + primary menu; the meaningless "toolbar style" preference dropped in GTK4 only). GTK3 is untouched apart from the sanctioned preferences addition.

### Verification

Everything compiles; the GTK3 sweep (23 modules) and a GTK4/libadwaita stub sweep (18 modules, including the window and app end-to-end) both import cleanly; the pure model/report tests pass; and the dispatcher + config round-trip are unit-tested.
