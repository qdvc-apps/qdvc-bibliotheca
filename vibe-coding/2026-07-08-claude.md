## 1. User request

Please help me write a Python 3 GTK 3 program called "QDVC Bibliotheca". Its purpose is to help the user, an academic (researcher/scholar/professor) or student, manage their personal collection of articles, papers and books.

Catalogue data for articles (including journal articles, conference papers, books, etc.) exist as the following files:

- An authoritative BibTeX (.bib) file, from which reference/bibliometric information is obtained. This represents the article as it exists "upstream" (from the publisher), with no information specific to the user.
- A Markdown file with a YAML frontmatter header, documenting the user's own relationship with the article, and the status of the article in the QDVC Bibliotheca catalogue:
    - YAML frontmatter:
        - Full-text PDF location (if available)
        - Full-text EPUB location (if available)
        - List of "my works" which reference this article
    - Markdown text after the YAML frontmatter represents the user's personal notes about the article, written in Markdown format.

These files are all stored in a "workspace" folder. Use File --> Open Workspace and File --> Close Workspace, the user can switch between multiple workspaces. The workspace stores the files like so:

- (workspace root)
    - bibtex
        - A
        - B
        - C
        - ...
        - Z
    - markdown
        - A
        - B
        - C
        - ...
        - Z
    - my_works
        - project1.yml
        - project2.yml
        - ...

The naming convention for both the bibtex file and the markdown file is as follows: AuthorSurnamesYear_suffix. This is the **Bibliotheca ID** (`bibliotheca_id`). For example:

    - SmithJones2025_MISQ.bib and SmithJones2025_MISQ.md
    - SmithEtAl2023_ISJ.bib and SmithEtAl2023_ISJ.md
    - Jones2009_JAIS.bib and Jones2009_JAIS.md

The author surnames follow an APA-ish (might not perfectly) pattern: single author -> just one name, two authors -> both names, more than two authors -> "et al". The suffix is for quick identification (in this example, MISQ, ISJ and JAIS are prominent journals in a particular academic discipline) and for disambiguation (e.g., Davison2026_ISJ_generative; Davison2024_ISJ_brevity).

As mentioned above, the user can also keep track of their own "my works". Each of these is simply a YAML file, with no particular naming convention at this stage. The YAML file tracks the list of articles (based on bibliotheca_id) cited in that work. If a particular instance of "my works" is a published paper, the YAML file also identifies the bibliotheca_id of the published version.

Any other program configuration is stored in a config.yml at an appropriate XDG config location.

The frontend of the program is organised as tabs:

1. Catalogue
    - This is organised as a three-pane master-detail view.
    - Pane 1 (sidebar):
        - All articles
        - By type <-- filters according to a specific type
            - Journal articles
            - Conference papers
            - Book chapters
        - My works <-- filters according to a specific project
            - Project 1
            - Project 2
            - Project 3
    - Pane 2 (master): it is table view of all the articles on record. An article could be a journal article, conference paper, book, book chapter, webpage, or other source type.
    - Pane 3 (detail):
        - A read-only textbox with the reference of the article (APA 7 format), based on the bibtex file. There should be 2 "copy" buttons: one to copy the reference as-is (e.g., with italics on journal names); and another to copy the reference as plaintext (all rich formatting stripped away).
        - A read-and-write textbox for the markdown file

2. DOI Lookup
    - This is a simple screen with just a text-entry box, a button called "Lookup", and an label below all that for status information
    - If the user-entered text happens to be, verbatim, a DOI on file: then the "Lookup" button will take the user to that file
    - If not: the label (status information) has the following kind of text:
        "Sorry, no records found for DOI = 10.1234/example"
